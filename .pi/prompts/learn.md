# Learn

Revisá la sesión y los cambios recientes con foco en: $ARGUMENTS

Objetivo: convertir aprendizajes útiles en conocimiento persistente y versionado
del repo.

Buscá:

- Reglas nuevas de trabajo que deberían ir a `AGENTS.md`.
- Comandos repetibles que deberían documentarse en `README.md`.
- Investigaciones o decisiones que deberían guardarse en `docs/research/`.
- Prompts reutilizables que deberían agregarse en `.pi/prompts/`.
- Configuración local o de tooling que debería quedar documentada.

Antes de editar, proponé brevemente qué conviene persistir y dónde.

No guardes secretos, API keys ni datos privados. Si aparece información sensible,
recomendá moverla a archivos ignorados por git o variables de entorno.
