# Context7

Fecha: 2026-05-01

## Resumen

Context7 provee documentación actualizada para agentes y herramientas de
programación. Puede usarse de dos formas principales:

- CLI directo con `ctx7`.
- Servidor MCP con `@upstash/context7-mcp`.

## Configuración segura

No commitear API keys.

Este repo ignora archivos locales de entorno:

```text
.env
.env.*
```

Para configurar Context7 localmente:

```sh
cp .env.example .env.local
```

Luego editar `.env.local`:

```sh
CONTEXT7_API_KEY=ctx7sk-...
```

Para cargar la variable en la shell:

```sh
export CONTEXT7_API_KEY="ctx7sk-..."
```

## CLI

Buscar una librería:

```sh
npx ctx7 library leiningen "project.clj test repl"
```

Consultar documentación con el id devuelto:

```sh
npx ctx7 docs <library-id> "lein test dependencies"
```

Para scripting, usar `--json` cuando aplique.

## Setup automático

Context7 recomienda:

```sh
npx ctx7 setup
```

Para modo CLI:

```sh
npx ctx7 setup --cli
```

Para modo MCP:

```sh
npx ctx7 setup --mcp
```

Con API key explícita:

```sh
npx ctx7 setup --api-key "$CONTEXT7_API_KEY"
```

## MCP local

Ejecutar servidor MCP local por stdio:

```sh
npx -y @upstash/context7-mcp --api-key "$CONTEXT7_API_KEY"
```

También puede configurarse con variable de entorno:

```sh
CONTEXT7_API_KEY="$CONTEXT7_API_KEY" npx -y @upstash/context7-mcp
```

## MCP remoto

Endpoint remoto:

```text
https://mcp.context7.com/mcp
```

Header requerido si se usa API key:

```text
CONTEXT7_API_KEY: <api-key>
```

## Uso recomendado en este repo

1. Mantener la API key fuera de git.
2. Usar CLI para investigaciones puntuales.
3. Usar MCP si Cursor o pi quedan configurados para consumir Context7.
4. Documentar en `docs/research/` las investigaciones importantes.
