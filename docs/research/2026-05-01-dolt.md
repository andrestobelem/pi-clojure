# Dolt para chat Markdown

## Decisión exploratoria

Vamos a probar Dolt como base de datos SQL con versionado tipo Git para
conversaciones en Markdown.

Dolt es open source y permite trabajar con tablas usando conceptos familiares de
Git:

- commits;
- ramas;
- diffs;
- merges;
- push/pull;
- historial versionado.

## Hipótesis

Dolt puede ser útil para:

- versionar snapshots de salas;
- comparar cambios en mensajes editados;
- experimentar con ramas de conversaciones/documentos;
- exportar una sala Markdown como dataset versionado;
- auditar cambios usando commits.

No asumimos todavía que sea el mejor motor para tiempo real, presencia o
WebSockets de alto volumen.

## Prueba local

El entorno local usa la imagen oficial `dolthub/dolt-sql-server` mediante Docker
Compose. La base inicial se llama `chat_markdown`.

```sh
docker compose up -d dolt
```

Aplicar el esquema inicial:

```sh
docker compose exec -T dolt \
  dolt --data-dir /var/lib/dolt/chat_markdown sql < db/dolt/001_chat_markdown.sql
```

Crear un commit Dolt del esquema:

```sh
docker compose exec dolt sh -lc '
  cd /var/lib/dolt/chat_markdown &&
  dolt config --local --add user.name "pi-clojure" &&
  dolt config --local --add user.email "pi-clojure@example.local" &&
  dolt add . &&
  dolt commit -m "create chat markdown schema"
'
```

Conectarse por MySQL compatible:

```sh
mysql \
  --host 127.0.0.1 \
  --port 3307 \
  --user chat \
  --password=chatpass \
  chat_markdown
```

Ver tablas:

```sh
docker compose exec dolt sh -lc \
  'cd /var/lib/dolt/chat_markdown && dolt sql -q "show tables"'
```

Ver cambios pendientes:

```sh
docker compose exec dolt sh -lc \
  'cd /var/lib/dolt/chat_markdown && dolt status && dolt diff --stat'
```

Descartar cambios de prueba:

```sh
docker compose exec dolt sh -lc \
  'cd /var/lib/dolt/chat_markdown && dolt reset --hard'
```

## Resultado de la prueba

Se levantó el contenedor correctamente y quedó saludable con:

```sh
docker compose up -d dolt
docker compose ps
```

Se aplicó el esquema inicial y se creó un commit Dolt:

```text
create chat markdown schema
```

`show tables` devolvió:

```text
message_events
messages
rooms
users
```

También se insertaron datos mínimos de ejemplo, se verificó que `dolt diff --stat`
mostrara filas agregadas y luego se ejecutó `dolt reset --hard` para dejar la
base limpia.

## Aprendizajes operativos

- La opción `dolt sql --database chat_markdown` no está disponible en la CLI
  probada. El comando correcto dentro del contenedor es usar `--data-dir`:

  ```sh
  dolt --data-dir /var/lib/dolt/chat_markdown sql
  ```

- Antes de commitear en Dolt hay que configurar identidad local:

  ```sh
  dolt config --local --add user.name "pi-clojure"
  dolt config --local --add user.email "pi-clojure@example.local"
  ```

- Para esta exploración conviene usar credenciales locales no sensibles en
  `.env.example` y mantener valores reales en `.env.local` o variables de
  entorno ignoradas por git.

## Evaluación inicial

Dolt parece prometedor para versionado, diffs, commits y snapshots de salas o
documentos derivados de conversaciones Markdown. Todavía no validamos que sea el
motor adecuado para WebSockets, presencia o escritura en tiempo real de alto
volumen.

La hipótesis actual es usar Dolt primero como laboratorio Git-like para datos y
compararlo más adelante contra alternativas como PostgreSQL con event log o XTDB.

## Esquema inicial

El esquema mínimo está en `db/dolt/001_chat_markdown.sql` e incluye:

- `rooms`;
- `users`;
- `messages`;
- `message_events`.

`messages.body_markdown` es la fuente de verdad. `messages.body_html` queda como
campo derivado para HTML sanitizado.

`message_events` mantiene un log append-only de cambios para aproximarnos a un
modelo event-sourced además del versionado de Dolt.
