# Commit

Prepará commits atómicos a partir de los cambios actuales.

Pasos:

1. Inspeccioná `git status` y `git diff`.
2. Separá cambios por intención lógica.
3. Verificá tests o linters relevantes antes de commitear.
4. Creá commits usando Conventional Commits.
5. Mantené cada commit pequeño, coherente y reversible.

No mezcles documentación, configuración, refactors y features si no pertenecen
al mismo cambio lógico.
